Frequently Asked Questions

To add your profile card:

  1. Fork the repository on GitHub
  2. Clone your forked repository to your local machine
  3. Add your information to the projects.json file following the existing format
  4. Add your profile image to the images/ folder
  5. Commit your changes and create a pull request

Make sure to follow the contribution guidelines for the best chance of your PR being accepted.

Profile images should meet these requirements:

  • Format: JPG, PNG, or GIF
  • Size: Maximum 2MB
  • Dimensions: Square aspect ratio recommended (1:1)
  • Resolution: At least 200x200 pixels
  • Content: Professional headshot or avatar

Images will be automatically resized and optimized for the website.

To contribute a project:

  1. Create a new folder in the projects/ directory
  2. Add your project files (HTML, CSS, JS, etc.)
  3. Ensure your project works independently
  4. Test your project in different browsers
  5. Create a descriptive README.md for your project
  6. Submit a pull request with a clear description

All projects must be original, functional, and follow our code of conduct.

We accept a wide variety of web development projects including:

  • Interactive web applications
  • Games and animations
  • Utility tools and calculators
  • Creative experiments
  • Educational content
  • UI/UX demonstrations

Projects must be built with web technologies (HTML, CSS, JavaScript) and should be original work. Commercial projects or those with licensing restrictions may not be accepted.

Common issues and solutions:

  • Paths not working: Use relative paths from your project folder
  • External resources blocked: Host assets locally or use CDNs that allow cross-origin requests
  • JavaScript errors: Check browser console for errors and ensure modern JavaScript features are supported
  • Styling issues: Make sure CSS is properly linked and selectors are correct
  • Responsive design: Test on different screen sizes and use media queries

If issues persist, check our troubleshooting guides or submit a bug report.

CORS errors occur when trying to access external APIs or resources. Solutions:

  1. Use a proxy: Set up a local development server with CORS proxy
  2. JSONP (for APIs that support it): Use JSONP instead of fetch/XMLHttpRequest
  3. Host locally: Download and host external resources in your project folder
  4. CORS-enabled APIs: Use APIs that have CORS enabled or provide JSONP support
  5. Development server: Use tools like Live Server or http-server for local development

For production deployment, ensure your hosting platform supports the APIs you're using.

To update your profile:

  1. Fork the repository again if you haven't recently
  2. Locate your entry in projects.json
  3. Update your information (name, bio, links, etc.)
  4. Replace your profile image if needed
  5. Commit changes and create a pull request

Make sure to follow the same format as other entries. Your changes will be reviewed before being merged.

Yes, you can request to have your profile removed:

  • Create an issue on GitHub requesting removal
  • Provide your name and reason for removal
  • Include a link to your profile for verification
  • Wait for maintainer review and approval

We respect your privacy and will process removal requests promptly. Note that contributed projects may remain if they add value to the community.
